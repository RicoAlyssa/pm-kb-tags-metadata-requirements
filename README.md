# 知识库标签与元数据需求项目

本目录包含两个独立的六桶需求项目：

- `知识库新增标签/`
- `知识库新增元数据/`

云端仓库：

- `https://github.com/RicoAlyssa/pm-kb-tags-metadata-requirements`

每个项目由 `00_项目总脑.md` 路由，并包含产品定义、竞品调研、需求文档、
分析报告、设计素材和接口六个桶。

当前已确认的产品边界：

- 知识库级标签：用于知识库检索路由，并涉及项目/业务计费归属。
- 文档级标签：用于检索前过滤文档及切片，不涉及计费。
- 文档元数据：独立的自定义 key-value 原始信息，用于检索过滤；底层
  已有接口，当前重点是控制台暴露与前后端字段映射。
- 正式竞品范围：仅火山云/火山引擎与阿里云；Dify、RAGFlow、AWS 等
  仅可作为参考实现，不进入正式竞品横向结论。

所有接口事实以瀚海 OPEN-API/Apifox 的只读证据为准。项目文件不得保存
密码、Token、Cookie 或 API Key。
