## 二、已确认事实

1. 阿里云百炼截图中出现的是文档/数据级标签，不是知识库级计费标签。
2. 标签入口出现在两个位置：
   - 创建知识库第 2 步“选择数据”：上传文件后新增标签。
   - 已有知识库文档列表：单文档行内“更多 > 标签”。
3. 标签是轻量 chip 输入，不需要字段类型、字段描述、取值方法、枚举选项。
4. 标签输入区可见数量上限提示，截图中出现 `1/100`、`2/100`。
5. 已创建标签可复用，页面文案为“已创建标签（点击可添加至输入框）”。
6. 标签管理弹窗包含：
   - 新增标签输入框
   - 已创建标签列表
   - 取消 / 确定按钮
7. 阿里云在创建知识库页面右侧展示费用详情：知识库标准版 `0.03 元/小时`、收费服务 `1 个`。该截图只能证明知识库资源有计费信息，不能证明“标签本身参与计费”。

## 三、对我们标签需求的启发

| 标签类型 | 阿里云证据 | 我们 PRD 中建议落法 |
| --- | --- | --- |
| 文档级标签 | 有明确入口：创建时给上传数据打标签；已有文档可后置管理标签。 | 文档级标签应覆盖上传前/上传后两个场景，并支持批量操作。 |
| 标签输入方式 | chip 输入 + Enter 确认 + 已有标签点击复用。 | 保留轻量输入，不做复杂 schema；重点做去重、数量上限、搜索/复用。 |
| 文档级标签用途 | 截图未直接证明检索过滤，但会议纪要中明确文档标签用于检索前过滤和权限隔离。 | 文档级标签 PRD 中应写清用于文档切片检索过滤，不涉及计费。 |
| 知识库级标签 | 阿里云截图未证明存在知识库级标签。 | 不能照搬阿里云；知识库级标签要按我们的业务平台/计费分账要求单独设计。 |
| 计费关系 | 阿里云展示知识库资源费用，但标签不直接计费。 | PRD 中必须明确：知识库级标签涉及计费/分账；文档级标签不涉及计费。 |

## 四、待确认项

1. 阿里云是否还有独立的知识库级标签入口；当前截图只证明文档/数据级标签。
2. 标签是否支持批量添加、批量删除、跨文档批量复用。
3. 标签是否可作为知识检索 API 的过滤条件，过滤语法如何表达。
4. 标签是否参与权限控制，还是仅作为检索筛选/管理分类。
5. 标签数量上限 `100` 是单文档上限、单知识库标签池上限，还是单次输入上限。
6. 标签重复、大小写、空格、中英文符号的校验规则。
7. 删除“已创建标签”后，对已关联文档是否同步移除。
