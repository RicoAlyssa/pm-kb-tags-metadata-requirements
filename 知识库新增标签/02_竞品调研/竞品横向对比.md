# 竞品横向对比：知识库新增标签

状态：初版（正式竞品已收敛为火山云、阿里云；控制台截图待采）

## 1. 范围说明

正式竞品只包含：

- 火山云/火山引擎：Viking 知识库、火山方舟、扣子关联火山知识库相关链路。
- 阿里云：百炼 Model Studio、PAI 知识库相关链路。

Dify、RAGFlow、AWS 等实现只保留为参考实现，不进入竞品横向结论。

## 2. 快速结论

- 火山云的标签更接近我方“文档级标签”：官方文档明确支持创建知识库时定义标签、原始文档页编辑标签，并在知识检索中按标签过滤文档范围；具体 API 字段名仍待 Network/HAR 确认。
- 火山云也暴露了计费强相关信息：火山知识库在扣子中使用时，存储、召回、重排计费由火山知识库侧收取，创建后上传文档会预留计算资源并开始计费；这对我方“知识库级标签涉及计费”有业务链路参考，但尚未证明火山标签本身直接承载成本中心。
- 阿里云百炼明确支持文档标签过滤：上传时设置、数据管理页编辑、API `tags` 参数、智能体应用控制台配置。
- 阿里云百炼还支持多知识库联合检索和知识库路由；这对我方“知识库级标签用于 AI/应用选择知识库”有参考价值，但阿里公开文档当前体现的是路由/权重/标签过滤组合，不是“知识库级标签计费归属”。
- 当前 PRD 要保留两条线：知识库级标签负责选库/业务归属/计费链路；文档级标签负责文档过滤/权限过滤，不进入计费。

## 3. 横向对比表

| 维度 | 我方目标 | 火山云/火山引擎 | 阿里云百炼/PAI | 产品判断 |
|---|---|---|---|---|
| 标签对象层级 | 知识库级标签、文档级标签分开 | 标签在创建知识库时定义，导入文档后给文档设置标签值；公开案例强调文档权限标签 | 文档搜索类知识库支持文档标签；检索服务支持多知识库绑定、知识库路由和标签过滤 | 我方仍需显式拆成知识库级/文档级，避免竞品混在一层 |
| 文档级标签 | 给文档打标，检索前过滤文档/切片，不涉及计费 | 明确可在“原始文档-标签”列编辑；知识检索参数支持按标签过滤文档范围 | 支持上传时设置标签、数据管理页编辑标签，API `tags` 指定标签 | 文档级标签 P0 证据充分，应进入 PRD 主链路 |
| 知识库级标签/选库 | 给知识库分类，用于 AI/应用选择知识库 | 火山知识库可被扣子应用/工作流关联；公开文档更多体现知识库资源和文档标签 | 知识检索服务支持最多 15 个知识库联合检索，并可开启知识库路由 | 我方知识库级标签可结合“知识库路由”设计，但标签是否为选库主条件需继续采证 |
| 标签创建/编辑限制 | 需要定义创建、编辑、删除、变更生效规则 | 公开文档说明火山标签仅旗舰版支持，且创建知识库后不支持变更标签名称 | 阿里公开文档说明标签可在上传时设置、数据管理页编辑；更多限制待控制台采证 | 我方需要重点写“创建后是否可改、删除影响、生效时间” |
| 权限过滤 | 文档标签可作为过滤条件，但不能替代完整权限系统 | 火山案例直接用部门/角色标签实现权限相关过滤，检索阶段结合用户身份过滤 | 阿里公开文档强调标签过滤提升检索准确性，未直接证明标签承担权限 | 我方 PRD 应写“标签过滤和鉴权分层，权限优先” |
| 计费/业务归属 | 只有知识库级标签涉及计费；文档级标签不涉及计费 | 火山知识库资源本身会产生存储/召回/重排/模型等费用；创建后上传文档即预留资源并开始计费 | 百炼知识库有商业化和配额/存储限制；标签过滤本身是否计费待确认 | 我方不能把文档级标签写进计费；知识库级标签需对接业务平台/计费归属 |
| 检索配置 | 支持标签条件进入召回链路 | 火山知识检索参数支持标签过滤文档范围；字段名待 HAR 确认 | 阿里 API 可用 `tags`，检索服务独立配置含标签过滤 | 我方接口需明确知识库级选库条件和文档级标签过滤条件的组合规则 |
| 控台入口 | 文档列表/详情、知识库设置、应用/检索配置 | 火山：创建知识库高级配置、原始文档标签列、检索/问答配置 | 阿里：上传文档、数据管理页、智能体应用配置、检索服务配置 | 后续 Pencil 原型需要覆盖这些状态 |

## 5. 对 PRD 的建议

1. 知识库级标签：不能照抄火山/阿里文档级标签，应围绕“选库 + 业务归属 + 计费链路”单独定义。
2. 文档级标签：可以强参考火山和阿里的文档标签过滤能力，但明确“不计费、不替代权限系统”。
3. 检索链路：需要同时支持知识库级选库条件和文档级标签过滤条件，并明确组合顺序。
4. 计费链路：竞品证明知识库资源会计费，但还没有证明标签字段本身等于计费归属；我方需业务平台确认。
