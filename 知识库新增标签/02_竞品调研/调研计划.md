# 调研计划：知识库新增标签

状态：执行中（公开官方文档证据已补；控制台截图/Network 证据待 Playwright 采集）

更新日期：2026-07-10

## 1. 调研目标

本需求必须分成两条线调研，不能把“标签”写成一个含混功能：

| 线 | 目标 | 为什么影响当前需求 |
|---|---|---|
| 知识库级标签 | 调研竞品是否支持给知识库打标签、按标签筛选/路由知识库、标签 API、绑定关系、使用量、删除影响 | 我方知识库级标签承担 AI/应用选择知识库，并且涉及项目/业务/计费归属 |
| 文档级标签 | 调研竞品是否支持给文档打标签、批量打标、按标签做检索前过滤、权限边界、删除/改名生效 | 我方文档级标签用于过滤文档及切片，不进入计费链路 |

## 2. 当前已确认边界

- `VERIFIED`：知识库级标签与文档级标签必须分开设计。
- `VERIFIED`：只有知识库级标签涉及计费；文档级标签不涉及计费。
- `VERIFIED`：文档级标签用于检索前过滤文档/切片。
- `TO_CONFIRM`：底层是否已有我方标签接口、字段、限额与检索表达式。
- `TO_CONFIRM`：知识库级标签的计费归属模型由业务平台还是知识库控制台主导。

## 5. 采证动作

### 5.1 知识库级标签

1. 进入知识库列表/知识库详情/标签管理入口。
2. 新建、重命名、删除标签，记录名称规则、重复名、长度、颜色/描述等。
3. 给知识库绑定/解绑多个标签，记录批量能力和使用量。
4. 在知识库列表或应用选择知识库时按标签筛选。
5. 查是否出现项目、部门、成本中心、用量、账单、费用归属等字段。
6. 若页面行为依赖接口，采集已脱敏 Network/HAR；不点击真实“发送/调试”。

### 5.2 文档级标签

1. 在文档列表、文档详情、批量操作入口查找打标能力。
2. 记录单文档/批量打标、删除标签、修改标签后的文档影响。
3. 在召回测试、应用编排或检索 API 中配置标签过滤条件。
4. 记录权限关系：标签只是过滤条件，还是直接参与授权。
5. 记录异常：无标签、标签不存在、重复标签、删除被引用标签、索引未生效。
