# 评审记录：知识库新增标签

## 2026-07-14 v0.5 用户当前版同步审查

结论：当前 PRD 可进入产品/前后端/QA 初评，但不能作为最终排期闭环。已同步用户当前 PRD 中最容易被误解的边界：标签需求拆成知识库级标签与文档级标签；新建知识库页只允许选择知识库级标签，文档级标签不出现在创建表单；只有知识库级标签涉及计费。

| 维度 | 审查结论 | 后续动作 |
|---|---|---|
| 产品范围 | 知识库级标签、文档级标签已分离；文档级标签不承载计费 | 保持两个对象在 PRD、原型、接口和验收中分开表达 |
| 创建链路 | 已补 `TAG-KB-008` 与 `AC-TAG-008`，明确新建知识库页为可选知识库级标签选择，不阻断现有创建/购买链路 | 原型只在“所属项目”后增加知识库级标签选择，不增加文档级标签入口 |
| 计费边界 | 已确认只有知识库级标签涉及计费；文档级标签不得出现计费字段 | 业务平台/计费需确认主数据、同步方向、失败补偿和审计 |
| 接口 | 当前星流与瀚海未发现独立标签 CRUD/绑定/检索过滤接口 | 后端需给出标签定义、绑定、检索过滤、IAM Action、错误码和索引生效语义 |
| 检索 | 文档级标签用于检索前缩小文档/切片候选范围；不得绕过权限 | 检索负责人需确认 AND/OR、包含/不包含、权限组合顺序和生效延迟 |
| 设计 | 现有 HTML 最小改造原型可作为评审讨论稿；正式 Pencil 原型仍待最终确认 | 原型不得把标签与元数据混在同一张图里 |

### 当前阻塞项

1. 知识库级标签的计费归属模型、业务平台主数据源、同步方向、失败补偿机制未确认。
2. 金山云资源标签/项目能力能否直接绑定星流知识库资源仍未确认。
3. 标签 CRUD、绑定/解绑、检索过滤和 IAM Action 尚无正式接口契约。
4. 标签变更后检索索引的生效时间、失败状态和重试机制未确认。
