# PRD 补充：火山方舟标签调研对需求的影响

更新时间：2026-07-10  
适用 PRD：知识库新增标签

## 1. 必须进入 PRD 正文的结论

1. 知识库级标签与文档级标签必须分开。
   - 知识库级：参考火山“费用标签”，位于知识库详情顶部基础信息区，文案明确服务于“项目分账”。
   - 文档级：参考火山“文档标签”，位于创建知识库高级配置、原始文档列表、检索/问答筛选范围中，不涉及计费。

2. 知识库级标签是计费/业务归属标签。
   - 字段形态：标签名 + 标签内容。
   - 输入方式：选择已有标签或输入。
   - 需要展示业务平台/计费系统同步状态、失败原因、最近同步时间、审计记录。
   - 删除或修改时必须提示计费影响与应用引用影响。

3. 文档级标签是检索过滤 Schema。
   - 字段形态：标签名 + 字段类型 + 标签选项（选填）。
   - 火山上限：添加标签 `1/180`，可作为容量参考，不直接照抄。
   - 字段类型参考：`List<int64>`、`List<string>`、`String`、`Int64`、Float32/小数、`Bool`、`Date_time`、`Geo_point`。
   - 示例参考：所属行业（List<string>，医疗/游戏/教育）、优先级（Float32）、是否公开（Bool，是/否）。

4. 文档级标签检索过滤必须进入召回链路。
   - 知识检索：火山用输入框 `@` 快速指定筛选范围。
   - 知识问答：火山用右侧“筛选范围”常驻面板。
   - 我方建议抽象为统一组件：选择文档、选择目录、按标签筛选。

5. `doc_id` 是内置可过滤字段。
   - 火山在文档标签编辑弹窗和问答筛选里都体现 `doc_id` 可过滤。
   - 我方标签 PRD 中可以把 `doc_id` 作为系统内置筛选字段，不要求用户作为标签配置。

## 2. 建议改写的 PRD 模块

| PRD 模块 | 需要补充的内容 |
|---|---|
| 背景 | 加一句：标签需求有两条互不合并的产品线，知识库级标签解决选库/业务归属/计费，文档级标签解决文档过滤/权限前置筛选 |
| 信息架构 | 增加三个入口：知识库详情顶部费用标签、创建知识库高级配置文档标签、检索/问答筛选范围 |
| 功能需求：知识库级标签 | 增加“标签名 + 标签内容”“项目/成本/业务归属”“同步状态”“计费影响提示” |
| 功能需求：文档级标签 | 增加 Schema 定义：标签名、字段类型、选项；增加容量、类型、枚举、空值规则 |
| 检索链路 | 增加筛选范围组件：选择文档、选择目录、按标签筛选；条件支持 AND/OR、运算符、输入值 |
| 交互截图表 | 增加火山采证截图作为竞品参考，不作为我方最终原型 |
| 验收标准 | 增加：知识库级标签变更不会影响文档标签；文档标签过滤不会改变计费归属 |

## 3. 我方交互建议

| 页面 | 区域 | 交互建议 |
|---|---|---|
| 知识库详情页 | 基础信息区 | 展示“知识库级标签/费用归属标签”；点击编辑打开 key-value 弹窗；保存前提示计费/业务平台影响 |
| 标签管理页 | 知识库级标签 Tab | 管理业务/计费标签，展示绑定知识库数、计费同步状态、引用应用数 |
| 标签管理页 | 文档级标签 Tab | 管理文档标签 Schema，展示字段类型、可选项、使用文档数、是否可删除 |
| 创建知识库页 | 高级配置 | 允许预置文档标签 Schema；不在这里配置知识库级费用标签，避免创建链路过重 |
| 原始文档列表 | 标签列 | 展示文档标签值；支持单文档编辑、表头筛选、批量打标 |
| 检索/问答调试 | 筛选范围 | 统一支持文档、目录、文档标签过滤；条件行包含字段、运算符和值 |

## 4. 暂不直接照抄火山的点

- 火山把强类型文档字段仍叫“标签”，我方如果同时做元数据，不能直接沿用这个命名，否则用户会分不清标签和元数据。
- 火山知识检索和知识问答的筛选入口形态不同；我方建议组件统一，只在布局上适配不同页面。
- 火山创建页名称提示“不支持中文”是其自身规则，我方不应默认继承，需以后端命名规则为准。
