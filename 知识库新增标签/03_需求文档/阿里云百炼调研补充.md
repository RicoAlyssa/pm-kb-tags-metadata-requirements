# 阿里云百炼调研补充：标签需求影响

## 一句话结论

阿里云百炼截图证明了“文档/数据级标签”的成熟交互：创建知识库上传数据时打标签，已有文档在行内菜单后置管理标签。但截图没有证明阿里云存在“知识库级计费标签”，所以我们的知识库级标签不能照搬阿里云，需要结合业务平台和计费分账单独设计。

## 对 PRD 的补充建议

| 标签类型 | 建议 |
| --- | --- |
| 文档级标签 | 作为检索前过滤、权限隔离、文档归类使用，不涉及计费。 |
| 知识库级标签 | 作为知识库选择、业务域路由、计费/分账维度使用，需要和业务平台标签体系对齐。 |
| 创建时标签 | 上传文档时允许直接新增标签；支持输入、回车确认、删除 chip、点击已有标签复用。 |
| 后置管理 | 文档列表需要单文档“标签管理”入口，也需要批量管理入口。 |
| 标签池 | 已创建标签应该形成知识库内标签池，便于复用；是否跨知识库共享需评审。 |
| 数量限制 | 阿里云可见上限提示为 100；我们需要明确单文档、单知识库、单次操作分别的上限。 |
| 计费提示 | 知识库级标签涉及计费时，界面需要显式提示“影响计费/分账/业务归属”；文档级标签不展示计费提示。 |

## 需要进入评审的问题

1. 知识库级标签由我们控制台维护，还是完全从业务平台同步。
2. 文档级标签是否也要接权限系统，还是只作为检索过滤条件。
3. 批量标签管理是否进入首版。
4. 标签删除是否需要二次确认，以及删除后对历史检索过滤条件的影响。
5. 检索 API 中标签过滤与元数据过滤是否共用一套表达式。
