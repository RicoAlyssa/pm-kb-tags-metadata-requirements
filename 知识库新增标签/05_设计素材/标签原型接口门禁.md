# 标签原型接口门禁

更新时间：2026-07-10  
状态：P0、P1 HTML 交互原型已完成；接口字段仍以评审结果为准，P2 不纳入本版。

## 已验证

- 瀚海 OPEN-API 当前可见目录未发现标签接口。
- 金山云当前 API Explorer / 星流平台知识库接口未发现知识库级标签、文档级标签、标签绑定/解绑、标签过滤字段。
- 知识库级标签涉及计费/业务归属；文档级标签不涉及计费。

## 原型约束

| 对象 | 能画什么 | 不能画什么 |
|---|---|---|
| 知识库级标签 | 作为业务平台/计费归属接入后的占位方案、同步状态、错误态 | 不写成本地控制台可直接创建并生效；不和文档级标签共用接口 |
| 文档级标签 | 在现有导入文档页赋值，并支持文档列表展示、单文档和批量打标 | 不进入新建知识库表单；不出现计费字段；不假设当前平台已有打标接口 |
| 检索过滤 | 描述未来需要文档级标签过滤参数 | 不写成当前 `RetrieveKnowledge` 已支持标签过滤 |

## 接口评审约束

- 文档标签导入、单篇更新、批量更新和检索过滤的具体字段保持 `TO_CONFIRM`。
- 知识库级标签必须等待业务平台/计费接口确认。
- Pencil 文件建议单独命名为 `kb-tags-prototype.pen`，不要使用早期 `kb-tags-metadata-light-prototype.pen`。
