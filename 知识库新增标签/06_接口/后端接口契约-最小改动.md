# 后端接口契约：知识库新增标签（最小改动方案）

状态：评审稿。
原则：知识库级标签和文档级标签分开；文档级标签不承载计费；知识库级标签涉及业务/计费归属，必须与业务平台同步；检索过滤在当前 `RetrieveKnowledge` 的 `RetrievalModel` 下兼容扩展。

## 1. 已核查的当前平台基线

来源：金山云 API Explorer，只读抓取，2026-07-14。

| 当前 Action | Version | 当前能力 | 与标签需求关系 |
|---|---:|---|---|
| `ImportDocuments` | `2025-11-14` | 创建知识库文档 | 可兼容增加可选 `DocumentTagIds`，用于导入时给本批文档打标 |
| `DescribeDocuments` | `2025-11-14` | 文档列表 | 建议可选返回文档标签 |
| `DescribeDocument` | `2025-11-14` | 文档详情 | 建议可选返回文档标签 |
| `RetrieveKnowledge` | `2025-11-14` | 知识库检索 | 建议在 `RetrievalModel` 下新增 `TagFilter` |
| `DeleteDocument` / `ReindexDocuments` / `ModifyDocumentStatus` | `2025-11-14` | 文档管理 | 标签变更触发索引刷新时可复用索引状态语义 |

当前平台 API Explorer 未发现标签 CRUD、标签绑定/解绑或标签过滤字段。因此标签能力需要新增 Action 和兼容扩展。

## 2. Action 适配总览

统一入口保持当前平台风格：

```text
POST http://aicp.api.ksyun.com/?Action=<ActionName>&Version=2025-11-14
Content-Type: application/json
```

## 3. 统一对象模型

### 3.1 KnowledgeBaseTag

| 字段 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| `TagId` | String | 是 | 知识库级标签 ID。 |
| `Name` | String | 是 | 标签名称，同租户/空间下唯一。 |
| `BusinessOwner` | Object | 是 | 业务/计费归属。 |
| `BusinessOwner.Type` | enum | 是 | `Project` / `Department` / `CostCenter`。如业务平台有其他类型，后端可扩展。 |
| `BusinessOwner.Id` | String | 是 | 业务平台对象 ID。 |
| `BusinessOwner.Name` | String | 否 | 展示名。 |
| `BoundDatasetCount` | Int | 是 | 已绑定知识库数。 |
| `BillingRelated` | Boolean | 是 | 固定为 `true`，用于区分文档级标签。 |
| `SyncStatus` | enum | 否 | `Synced` / `Pending` / `Failed`。前端常态不展示 `Synced`，仅展示 `Pending/Failed`。 |
| `SyncError` | String | 否 | 同步失败原因。 |
| `CreatedAt` / `UpdatedAt` | Long | 否 | 审计字段。 |

### 3.2 DocumentTag

## 4. 接口契约

### 4.1 `DescribeKnowledgeBaseTags`

用途：查询可选知识库级标签，支持新建知识库页、知识库基本信息编辑。

请求：

| 字段 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| `Keyword` | String | 否 | 标签名/业务归属模糊搜索。 |
| `DatasetId` | String | 否 | 传入时返回该知识库已绑定标记。 |
| `Page` / `Limit` | Int | 否 | 默认 1 / 100。 |

响应：

| 字段 | 类型 | 说明 |
|---|---|---|
| `RequestId` | String | 请求唯一标识。 |
| `Tags` | KnowledgeBaseTag[] | 标签列表。 |
| `BoundTagIds` | String[] | 传 `DatasetId` 时返回已绑定标签。 |
| `Total` / `HasMore` | Int/Boolean | 分页。 |

### 4.2 `CreateKnowledgeBaseTag`

请求：

| 字段 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| `Name` | String | 是 | 1-40 字符，同租户唯一。 |
| `BusinessOwner` | Object | 是 | 项目/部门/成本中心等计费归属。 |
| `ClientToken` | String | 否 | 幂等键。 |

响应：`Tag` + `RequestId`。

语义：创建后触发业务平台同步；若同步失败，标签保存成功但返回 `SyncStatus=Failed` 和 `SyncError`，前端只展示异常态。

### 4.3 `ModifyKnowledgeBaseTag`

请求：

| 字段 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| `TagId` | String | 是 | 知识库级标签 ID。 |
| `Name` | String | 否 | 新名称。 |
| `BusinessOwner` | Object | 否 | 新业务归属。 |
| `ClientToken` | String | 否 | 幂等键。 |

响应：`Tag` + `RequestId`。

语义：修改业务归属必须写操作审计；涉及计费归属变更时触发同步事件。

### 4.4 `DeleteKnowledgeBaseTag`

请求：`TagId`、`ClientToken`。

删除规则：

- 已绑定知识库、已发布应用或计费对象引用时，默认返回 `409 KnowledgeBaseTagInUse`。
- 不支持前端静默强删。若后续需要迁移/解绑，应单独设计 `MigrateKnowledgeBaseTag`，不放 P0。

### 4.5 `ModifyKnowledgeBaseTagBindings`

用途：给一个或多个知识库绑定/解绑知识库级标签。

请求：

| 字段 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| `Operation` | enum | 是 | `Add` / `Remove` / `Set`。 |
| `DatasetIds` | String[] | 是 | 知识库 ID 列表。P0 前端通常只传一个。 |
| `TagIds` | String[] | 是 | 标签 ID 列表。 |
| `ClientToken` | String | 否 | 幂等键。 |

响应：

| 字段 | 类型 | 说明 |
|---|---|---|
| `SucceededItems` | Object[] | `DatasetId`、`TagIds[]`。 |
| `FailedItems` | Object[] | `DatasetId`、`TagId`、`Code`、`Message`。 |
| `SyncStatus` | enum | `Pending/Failed/Synced`，仅异常态展示。 |

批量语义：部分成功，失败明细回传。

### 4.6 `DescribeDocumentTags`

用途：知识库设置内维护文档标签池。

请求：

| 字段 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| `DatasetId` | String | 是 | 知识库 ID。 |
| `Keyword` | String | 否 | 标签名搜索。 |
| `Page` / `Limit` | Int | 否 | 默认 1 / 100。 |

响应：`Tags: DocumentTag[]`、`Total`、`HasMore`、`RequestId`。

### 4.7 `CreateDocumentTag`

请求：

| 字段 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| `DatasetId` | String | 是 | 知识库 ID。 |
| `Name` | String | 是 | 1-40 字符，同知识库唯一。 |
| `ClientToken` | String | 否 | 幂等键。 |

响应：`Tag: DocumentTag`。

### 4.8 `ModifyDocumentTag`

请求：`DatasetId`、`TagId`、`Name`、`ClientToken`。

语义：只改名称；不含字段类型、取值方法、模型回复开关。

### 4.9 `DeleteDocumentTag`

请求：`DatasetId`、`TagId`、`ClientToken`。

删除规则：

- 若 `FilterReferenceCount > 0`，返回 `409 DocumentTagInUse`，要求先移除检索配置引用。
- 若仅绑定文档但无检索配置引用，允许删除；后端移除文档绑定并触发索引刷新。

### 4.10 `BatchModifyDocumentTags`

用途：单篇或批量添加/移除文档标签。

请求：

| 字段 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| `DatasetId` | String | 是 | 知识库 ID。 |
| `Operation` | enum | 是 | `Add` / `Remove` / `Set`。P0 UI 使用 `Add/Remove`。 |
| `DocumentIds` | String[] | 是 | 文档 ID 列表。 |
| `TagIds` | String[] | 是 | 文档级标签 ID。 |
| `ClientToken` | String | 否 | 幂等键。 |

响应：

| 字段 | 类型 | 说明 |
|---|---|---|
| `RequestId` | String | 请求唯一标识。 |
| `SucceededItems` | Object[] | `DocumentId`、`TagIds[]`。 |
| `FailedItems` | Object[] | `DocumentId`、`TagId`、`Code`、`Message`。 |
| `IndexBatch` | String | 触发索引刷新时返回，可复用索引状态轮询。 |

批量语义：部分成功，成功项保留。

### 4.11 `ImportDocuments` 扩展

请求体新增可选字段：

| 字段 | 类型 | 默认 | 说明 |
|---|---|---|---|
| `DocumentTagIds` | String[] | `[]` | 导入成功后的全部文档绑定这些文档级标签。 |

语义：

- 不传时与当前行为一致。
- 标签不存在/无权限时，导入请求应整体失败，避免出现“文档导入成功但标签丢失”的误解。
- 若后端希望导入成功、标签失败部分回传，需要 PRD 增加失败明细；当前原型不建议这样做。

### 4.12 `DescribeDocuments` / `DescribeDocument` 扩展

请求新增：

| 字段 | 类型 | 默认 | 说明 |
|---|---|---|---|
| `IncludeDocumentTags` | Boolean | `false` | 是否返回文档标签。 |

响应新增：

| 字段 | 类型 | 说明 |
|---|---|---|
| `DocumentTags` | DocumentTag[] | 文档已绑定标签。 |

兼容性：默认不返回，不影响旧调用。

### 4.13 `RetrieveKnowledge` 扩展

请求在现有 `RetrievalModel` 下新增：

```json
{
  "DatasetId": "kb_xxx",
  "Query": "退款流程",
  "RetrievalModel": {
    "TopK": 10,
    "TagFilter": {
      "IncludeTagIds": ["tag_after_sales", "tag_internal"],
      "ExcludeTagIds": ["tag_expired"],
      "IncludeMatchMode": "all"
    }
  }
}
```

字段：

| 字段 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| `IncludeTagIds` | String[] | 否 | 结果必须包含的文档级标签。 |
| `ExcludeTagIds` | String[] | 否 | 结果不得包含的文档级标签。 |
| `IncludeMatchMode` | enum | 否 | `all` / `any`，默认 `all`。 |

响应建议新增：

| 字段 | 类型 | 说明 |
|---|---|---|
| `FilterSummary` | Object | `BeforeDocumentCount`、`AfterDocumentCount`、`AppliedTagFilter`。 |
| `Records[].Document.DocumentTags` | DocumentTag[] | 命中文档标签摘要。 |

权限语义：文档权限先收敛可见范围，标签过滤只能继续缩小候选集，不能扩大权限。

## 5. 权限 Action 建议

| 权限 Action | 对应接口 |
|---|---|
| `aicp:DescribeKnowledgeBaseTags` | 查询知识库级标签 |
| `aicp:ManageKnowledgeBaseTags` | 创建/编辑/删除知识库级标签、绑定知识库 |
| `aicp:DescribeDocumentTags` | 查询文档级标签 |
| `aicp:ManageDocumentTags` | 创建/编辑/删除文档级标签 |
| `aicp:ModifyDocumentTags` | 单篇/批量文档打标 |
| `aicp:RetrieveKnowledge` | 标签过滤沿用检索权限 |

知识库级标签还需业务平台/费用权限校验；文档级标签不检查计费权限。

## 6. 错误模型

| HTTP | Code | 触发 | 前端处理 |
|---:|---|---|---|
| 400 | `InvalidParameter` | 名称为空、标签数组为空、Operation 非法 | 字段级错误 |
| 403 | `Forbidden` | 无标签管理/文档打标/费用权限 | 隐藏或禁用操作，弹权限提示 |
| 404 | `DatasetNotFound` / `DocumentNotFound` / `TagNotFound` | 对象不存在 | 刷新列表 |
| 409 | `Conflict.TagDuplicated` | 标签重名 | 定位名称输入 |
| 409 | `Conflict.DocumentTagInUse` | 文档标签被检索配置引用 | 阻断删除 |
| 409 | `Conflict.KnowledgeBaseTagInUse` | 知识库级标签被知识库/应用/计费对象引用 | 阻断删除 |
| 409 | `Conflict.BillingSyncFailed` | 业务平台同步失败 | 标签保存成功但展示异常态和重试 |
| 200 | `PartialFailure` | 批量绑定/解绑部分失败 | 展示失败清单，成功项保留 |
| 202/200 | `IndexRefreshing` | 文档标签变更触发索引刷新 | 成功轻提示；仅异常/处理中时展示 |

## 7. 仍需业务/后端拍板但不再空着的点

| ID | 问题 | 建议默认值 |
|---|---|---|
| TAG-CONFIRM-001 | 知识库级标签主数据源是谁 | 业务平台为主，星流保存绑定关系和展示缓存 |
| TAG-CONFIRM-002 | 业务归属类型 | P0 支持 `Project/Department/CostCenter` |
| TAG-CONFIRM-003 | 文档级标签数量上限 | 每知识库 100 个；每文档最多 20 个 |
| TAG-CONFIRM-004 | 标签名称长度 | 1-40 字符，同作用域唯一 |
| TAG-CONFIRM-005 | 批量最大文档数 | 100，与当前分页/导入量级一致 |
| TAG-CONFIRM-006 | 检索生效 SLA | 目标 1 分钟内；超时返回/展示异常态 |
