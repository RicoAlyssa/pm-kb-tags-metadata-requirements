# 竞品接口调研：知识库新增标签

状态：进行中（2026-07-14 已做官方后端文档复核，未发起真实请求）
正式竞品：火山云/火山引擎、阿里云/百炼/PAI  
边界：Apifox/瀚海 OPEN-API 是我方底层接口源；金山云 API Explorer 是当前星流平台开放接口源；正式竞品只记录火山云/火山引擎、阿里云/百炼/PAI 的官方 API、官方文档与授权控制台/HAR 证据。

## 1. 结论先行

1. 标签接口不能只看我方 Apifox。
   - 瀚海 OPEN-API 用来确认“底层有没有接口、字段是什么、能不能复用”。
   - 金山云 API Explorer 用来确认“当前星流平台是否已开放”。
   - 火山/阿里用来确认“行业里文档标签、知识库路由、检索过滤 API 怎么表达”。

2. 标签必须继续拆成两个接口对象。
   - 知识库级标签：我方需求中涉及业务/项目/成本归属和计费同步；竞品证据目前主要来自火山控制台“费用标签”，阿里未看到同等知识库级计费标签 API。
   - 文档级标签：竞品都有更明确的检索过滤语义；可以作为我方 P0。

3. 不把阿里的 `tags` 或火山的文档标签直接照搬成我方字段。
   - 阿里 `tags` 更偏文档/文件标签过滤。
   - 火山“文档标签”已接近强类型字段/元数据，需和我方“元数据”需求隔离。

## 3. 阿里云接口要点

### 3.1 知识检索服务

阿里云百炼知识检索服务支持单知识库或多知识库联合检索，最多绑定 15 个知识库；每个知识库可以展开独立检索参数。独立配置里有“标签过滤”，说明它把标签过滤放在检索配置层，而不是只放在文档管理页。

对我方影响：

- 知识库级标签用于“选哪些知识库参与检索”时，应落在应用/检索路由层。
- 文档级标签用于“某个知识库内筛哪些文档/切片”时，应落在检索过滤层。
- 两者不要共用一个 `tags` 参数，否则 RD 很难判断作用域。

### 3.2 Retrieve + SearchFilters

阿里 `SearchFilters` 的标签查询示例：

```json
{
  "searchFilters": [
    {
      "tags": ["A大学", "学生会主席"]
    }
  ]
}
```

已确认语义：

- `searchFilters` 包含一个或多个子分组。
- 子分组之间默认 AND，且不可更改。
- 标签数组内部是 OR。
- 标签查询只支持文档搜索、音视频搜索类知识库。

对我方影响：

- 文档级标签过滤需要明确 AND/OR 规则，不要只写“支持按标签过滤”。
- 如果我方要支持“标签 A 且标签 B”，接口需能表达分组或显式逻辑关系。
- 前端交互必须展示组合关系，否则用户无法预期召回范围。

## 4. 火山接口/控制台要点

火山当前最强证据来自授权控制台与独立 Playwright HAR：

- 知识库详情顶部存在“费用标签”，文案明确和项目分账相关。
- 文档标签在创建知识库高级配置中定义，支持强类型字段。
- 原始文档列表和检索/问答筛选都出现按标签过滤。
- 实测标准版创建请求 `CreateCollection` 未携带标签 Schema，标准版文档列表没有“标签”列；旗舰版才显示标签列及编辑入口。
- 知识检索先调用 `CreateSearchConfig` 保存检索参数，再调用 `GetKnowledgeServiceChat` 执行检索。

已确认的过滤请求核心结构：

```json
{
  "Filter": {
    "Relation": 0,
    "Conditions": [
      {
        "FieldName": "liststring",
        "FieldType": "list<string>",
        "Operator": 0,
        "Values": ["product"]
      }
    ]
  },
  "FilterMode": "tag"
}
```

UI 对应关系已实测：`Relation: 0` 为“且”，该 `List<string>` 条件的 `Operator: 0` 为“包含”。尚未逐项验证其他类型的操作符枚举值。

官方 API/文档证据：

- `https://docs.volcengine.com/docs/82379/1261891?lang=zh`：创建知识库时初始化标签；知识库详情的原始文档页面可编辑标签；标签配置后可在检索时过滤文档范围。
- `https://docs.volcengine.com/docs/82379/1353778?lang=zh`：知识检索参数说明支持按标签过滤文档范围；过滤条件包含/不包含/大于/小于/大于等于/小于等于，条件关系为且/或。
- `https://docs.volcengine.com/docs/82379/1511946?lang=zh`：API 列表包含文档 `update_meta`，可作为文档 meta 更新链路证据。

仍需补证：保存费用标签、旗舰版创建时的标签 Schema 请求、文档级标签赋值保存请求；检索过滤请求体已补齐。不要从官方 UI 描述直接推造未观察到的枚举值。

下一轮必须抓的火山接口证据：

| 场景 | 需要抓取的请求 | 用途 | 脱敏要求 |
|---|---|---|---|
| 编辑费用标签 | 保存费用标签前后的 Network/HAR | 确认知识库级标签 API、是否关联资源标签/计费平台 | 去除 Cookie、Authorization、租户 ID、真实标签值 |
| 创建文档标签 Schema | 创建知识库高级配置提交请求 | 确认文档标签字段类型、字段名、选项结构 | 去除账号、项目、地域与资源 ID |
| 编辑文档标签 | 原始文档“标签”列保存请求 | 确认文档级标签赋值 API | 去除文档内容、文档 ID 可伪造 |
| 检索/问答按标签筛选 | 已完成 `CreateSearchConfig` + `GetKnowledgeServiceChat` 采证；后续补齐其他类型操作符 | 确认各类型 Operator 枚举及“或”关系值 | 去除 Query 中敏感内容 |
