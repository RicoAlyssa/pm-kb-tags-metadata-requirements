# 错误码：知识库新增标签

状态：评审稿。当前平台未公开标签业务错误码，以下为标签接口建议业务错误模型，后端可调整 Code 命名但需保留语义。

统一错误体：

```json
{
  "RequestId": "req_xxx",
  "Code": "Conflict.DocumentTagInUse",
  "Message": "文档标签正在被检索配置引用",
  "Details": {}
}
```

| HTTP | Code | 触发条件 | 前端提示 | 恢复方式 |
|---:|---|---|---|---|
| 400 | `InvalidParameter` | 标签名称为空、Operation 非法、标签数组为空 | “请检查标签配置” | 修改后重试 |
| 400 | `InvalidTagScope` | 文档级标签接口传入知识库级标签，或反向传入 | “标签类型不匹配” | 重新选择标签 |
| 403 | `Forbidden` | 无标签管理、文档打标或业务/计费权限 | “你没有操作该标签的权限” | 申请权限 |
| 404 | `DatasetNotFound` | 知识库不存在或无权访问 | “知识库不存在或无权限” | 刷新/返回列表 |
| 404 | `DocumentNotFound` | 文档不存在或已删除 | “文档不存在或已被删除” | 刷新文档列表 |
| 404 | `TagNotFound` | 标签不存在或已删除 | “标签不存在，请刷新” | 刷新标签列表 |
| 409 | `Conflict.TagDuplicated` | 同作用域标签重名 | “标签名称已存在” | 修改名称 |
| 409 | `Conflict.DocumentTagInUse` | 文档标签被检索配置引用，删除被阻断 | “标签正在被检索条件使用” | 先移除引用 |
| 409 | `Conflict.KnowledgeBaseTagInUse` | 知识库级标签被知识库、应用或计费对象引用 | “该标签仍有关联对象，无法删除” | 先解绑/迁移 |
| 409 | `Conflict.BillingSyncFailed` | 知识库级标签业务平台同步失败 | “标签已保存，但计费同步失败” | 展示重试入口 |
| 413/400 | `LimitExceeded` | 标签数量、单文档绑定数、批量文档数超限 | “已超过标签数量限制” | 减少数量 |
| 200 | `PartialFailure` | 批量绑定/解绑部分失败 | “部分文档处理失败” | 展示失败清单 |
| 202/200 | `IndexRefreshing` | 文档标签保存成功但索引刷新中 | “标签已保存，检索生效中” | 等待或刷新；成功态不常驻展示 |
| 500/503 | `IndexRefreshFailed` | 索引刷新失败 | “标签已保存，但检索暂未生效” | 重试刷新或联系管理员 |

## 批量失败明细建议

```json
{
  "SucceededItems": [
    {"DocumentId": "doc_1", "TagIds": ["tag_after_sales"]}
  ],
  "FailedItems": [
    {
      "DocumentId": "doc_2",
      "TagId": "tag_after_sales",
      "Code": "Forbidden",
      "Message": "无该文档打标权限"
    }
  ]
}
```

前端验收：批量部分失败时，成功项保留；失败项展示文档、标签和原因。
