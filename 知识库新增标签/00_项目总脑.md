# 00_项目总脑

## 项目

- 名称：知识库新增标签
- 当前阶段：竞品调研 + PRD 草稿 + 接口补齐 + 原型草稿
- 当前状态：已按用户确认将正式竞品收敛为火山云和阿里云；Dify/RAGFlow/AWS 等仅作参考实现；已基于金山云当前知识库列表/详情页制作轻量 HTML 原型草稿；PRD 已同步到 v0.5；后端接口已按金山云 API Explorer 的 `Action + Version=2025-11-14` 风格补齐最小改动契约，明确新建知识库页只选择知识库级标签，不出现文档级标签或元数据入口
- 下一确认门禁：后端/业务平台评审知识库级标签的计费归属模型、业务主数据源、同步方向与失败补偿；后端评审文档级标签 Action 命名、批量上限、过滤语义与索引生效 SLA

## 范围

### 已确认

- 知识库级标签与文档级标签必须分开定义、设计和评审。
- 知识库级标签用于帮助 AI/应用在多个知识库之间路由检索目标。
- 知识库级标签必须承载计费相关的项目/业务归属，并对接业务/应用平台。
- 文档级标签用于检索前过滤文档及其切片，由知识库侧管理，不进入计费链路。

### 非目标

- 当前阶段不把标签直接等同于完整的身份认证、授权或计费系统。
- 未经证据验证，不假设底层已经存在标签接口。

## 2026-07-10 接口门禁更新

- VERIFIED：当前瀚海 OPEN-API 可见目录未发现标签接口。
- VERIFIED：金山云当前 API Explorer / 星流平台知识库接口未暴露知识库级标签、文档级标签、标签绑定/解绑或标签过滤字段。
- VERIFIED：知识库级标签涉及计费/业务归属；文档级标签不涉及计费。
- DECISION：正式标签原型不得与元数据原型合并；知识库级标签和文档级标签也必须分开写。
- DECISION：当时未拿到业务平台/后端标签接口前，标签 PRD 中接口字段保持 `TO_CONFIRM`，原型只能作为“产品建议态”，不能写成“当前平台已支持”；2026-07-14 已补齐建议 Action 契约，仍需后端/业务平台评审最终命名、限额和同步规则。
- IMPACT：之前的“标签+元数据轻量合并 HTML 草稿”仅保留为历史草稿，不再作为正式原型依据。

## 2026-07-14 后端接口契约补齐

- VERIFIED：已按当前星流知识库接口风格补齐标签最小 Action 契约，不再只列缺口。
- VERIFIED：现有 `ImportDocuments`、`DescribeDocuments`、`DescribeDocument`、`RetrieveKnowledge` 可通过可选字段扩展承载文档级标签，不需要打断现有上传/检索链路。
- DECISION：知识库级标签仍然单独建模，因为它涉及业务归属和计费；文档级标签只做文档/切片过滤，不进入计费链路。
- DECISION：检索侧最小改动是在现有 `RetrieveKnowledge.RetrievalModel` 内新增 `TagFilter`，而不是新增一套检索 Action。
