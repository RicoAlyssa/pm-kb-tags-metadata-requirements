# 接口清单：知识库新增元数据

状态：评审稿。完整字段、请求/响应和错误语义见 [后端接口契约-最小改动.md](后端接口契约-最小改动.md)。

## 1. 当前平台基线

来源：金山云 API Explorer，只读抓取，2026-07-14。

当前知识库相关 Action：

- `ImportDocuments`
- `DescribeDocuments`
- `DescribeDocument`
- `RetrieveKnowledge`
- `DeleteDocument`
- `ReindexDocuments`
- `ModifyDocumentStatus`
- `BatchDisplayStatus` / `DisplayStatus` / `IndexingStatus`

结论：

- 当前 `ImportDocuments` 不应增加元数据值。
- 当前 `DescribeDocument.Metadata` 是返回内容粒度 `all/only/without`，不是客户自定义元数据。
- 当前 `RetrieveKnowledge` 未暴露 metadata filter。
- 瀚海 OPEN-API 已有元数据定义、赋值和过滤底层能力，但尚未以星流平台 Action 形式暴露。

## 2. 最小新增/适配 Action

统一入口：

```text
POST http://aicp.api.ksyun.com/?Action=<ActionName>&Version=2025-11-14
```

| API ID | Action | 作用 | 改动类型 | 证据/来源 |
|---|---|---|---|---|
| META-API-001 | `CreateMetadataField` | 新建元数据字段 | 新增 Action | 瀚海 `/metadata` create |
| META-API-002 | `ModifyMetadataField` | 编辑字段名称 | 新增 Action | 瀚海 `/metadata/{metadata_id}` patch |
| META-API-003 | `DeleteMetadataField` | 删除字段定义 | 新增 Action | 瀚海 `/metadata/{metadata_id}` delete |
| META-API-004 | `DescribeMetadataFields` | 查询字段定义列表 | 新增 Action | 瀚海 `/metadata` list |
| META-API-005 | `BatchModifyDocumentMetadata` | 单篇/批量编辑文档元数据值 | 新增 Action | 瀚海 `/documents/metadata` |
| META-API-006 | `DescribeDocuments` | 可选返回元数据摘要 | 兼容扩展 | 当前星流文档列表 |
| META-API-007 | `DescribeDocument` | 可选返回单篇文档元数据值 | 兼容扩展 | 当前星流文档详情 |
| META-API-008 | `RetrieveKnowledge` | 在 `RetrievalModel` 下新增 `MetadataFilter` | 兼容扩展 | 当前星流检索 + 瀚海过滤 |

## 3. P0 字段和语义

| 能力 | P0 结论 |
|---|---|
| 字段类型 | 前端 P0 只开放 `String`；后端可预留 `Number/Boolean/DateTime/StringList` |
| 字段编辑 | 只允许改名称，不允许改类型 |
| 字段删除 | 被检索配置引用时阻断；仅有文档值时允许删除并同步删除值 |
| 文档赋值 | 单篇和批量统一走 `BatchModifyDocumentMetadata` |
| 清空值 | 使用 `Operation=Clear`，不复用 `null/空字符串/缺失字段` 表达清空 |
| 批量失败 | 部分成功，返回 `SucceededItems` 和 `FailedItems` |
| 检索过滤 | 扩展 `RetrievalModel.MetadataFilter`，使用字段 ID + 操作符 + 值 |
| 成功态 | 不常驻展示“已生效”；仅索引刷新中/失败时展示异常提示 |

## 4. 前端可实现性

| 前端区域 | 后端支撑 |
|---|---|
| 知识库设置 > 元数据定义列表 | `DescribeMetadataFields` |
| 新建字段 | `CreateMetadataField` |
| 编辑字段 | `ModifyMetadataField` |
| 删除字段 | `DeleteMetadataField` |
| 文档列表元数据摘要 | `DescribeDocuments.IncludeCustomMetadata=true` |
| 单文档编辑元数据 | `DescribeDocument.IncludeCustomMetadata=true` + `BatchModifyDocumentMetadata` |
| 批量编辑元数据 | `BatchModifyDocumentMetadata` |
| 召回测试元数据过滤 | `RetrieveKnowledge.RetrievalModel.MetadataFilter` |

## 5. 保留给后端评审的命名项

以下不是需求缺口，而是后端最终命名决策：

- Action 名称是否完全采用本文建议。
- `MetadataFilter` 字段是否使用 `MetadataId` 还是后端内部字段名。
- 过滤操作符枚举是否与瀚海保持一致，或由星流网关做映射。
