## 瀚海 OPEN-API 元数据接口

| 能力 | 方法/路径 | 已验证请求字段 | 已验证响应字段 | 备注 |
|---|---|---|---|---|
| 新增元数据定义 | `POST /knowledge/openapi/v2/datasets/{dataset_id}/metadata` | Path：`dataset_id`；Body：`type`、`name` | `id`、`type`、`name` | 类型枚举值仍待确认 |
| 更新元数据定义 | `PATCH /knowledge/openapi/v2/datasets/{dataset_id}/metadata/{metadata_id}` | Path：`dataset_id`、`metadata_id`；Body：`name` | `id`、`type`、`name` | 当前文档只暴露改名；`type` 是否不可改待确认 |

## 更新文档元数据

- 路径：
  `http://openapi-hanhai-dev.ai.kingsoft.com/knowledge/openapi/v2/datasets/{dataset_id}/documents/metadata`
- 路径参数：`dataset_id`，UUID，必填。
- 请求体：
  - `operation_data[]`
  - `operation_data[].document_id`
  - `operation_data[].metadata_list[]`
  - 示例中的元数据项含 `id`、`value`、`name`
- 已见响应：`200`（无响应体）、`400`、`401`、`403`、`404`。

## 金山云当前平台 API Explorer 核查

入口：`https://apiexplorer.ksyun.com/#/document/documentList/239/1246`

当前星流平台「模型应用 > 知识库」已公开的知识库接口包括：

| 能力 | Open API | 与元数据需求的关系 |
|---|---|---|
| 查询模型列表 | `DescribeKnowledgeBaseModels` | 无元数据定义/赋值字段 |
| 开通知识库服务 | `ActivateKnowledgeBaseService` | 无元数据定义/赋值字段 |
| 知识库检索 | `RetrieveKnowledge` | 当前参数含 `DatasetId`、`Query`、`RetrievalModel`、`TopK`、`ScoreThreshold` 等；未见 metadata filter 字段 |
| 获取切片详情 | `DescribeChunk` | 无元数据定义/赋值字段 |
| 文档索引/嵌入状态 | `BatchDisplayStatus`、`DisplayStatus`、`IndexingStatus` | 无元数据定义/赋值字段 |
| 文档管理 | `ImportDocuments`、`DescribeDocuments`、`DescribeDocument`、`DeleteDocument`、`ReindexDocuments`、`ModifyDocumentStatus` | `DescribeDocument` 有 `Metadata` 参数，但语义是返回内容粒度 `all / only / without`，不是自定义元数据字段管理 |

结论：截至 2026-07-10 只读核查，金山云当前平台 API Explorer 未暴露“新增/更新/删除/查询元数据定义”和“批量更新文档元数据值”的接口。产品上可以判断为：瀚海底层能力已存在，但尚未以同等能力上线到当前星流平台开放接口/控制台；是否已有灰度或内部适配，需要后端确认。

## 参考实现：Dify Metadata API

> 仅作为非竞品参考实现，不等同于正式竞品证据，也不等同于我方字段。

| 能力 | 方法/路径 | 关键字段/语义 | 证据 |
|---|---|---|---|
| 创建 Metadata Field | `POST /datasets/{dataset_id}/metadata` | `name`、`type`；类型公开为 `string/number/time` | `https://docs.dify.ai/en/api-reference/metadata/create-metadata-field` |
