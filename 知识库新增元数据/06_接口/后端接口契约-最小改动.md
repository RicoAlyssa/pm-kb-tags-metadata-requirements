# 后端接口契约：知识库新增元数据（最小改动方案）

状态：评审稿。
原则：不改现有知识库创建链路；不把元数据值塞进 `ImportDocuments`；优先复用瀚海底层语义，并按当前星流平台 `Action + Version` OpenAPI 风格暴露。

## 1. 已核查的当前平台基线

来源：金山云 API Explorer，只读抓取，2026-07-14。

| 当前 Action | Version | 当前能力 | 与元数据需求关系 |
|---|---:|---|---|
| `ImportDocuments` | `2025-11-14` | 创建知识库文档；请求体包含 `DatasetId`、`Data`、`AddType`、`Ks3Path` | 不增加元数据值字段，避免导入阶段复杂化 |
| `DescribeDocuments` | `2025-11-14` | 文档列表；请求体包含 `DatasetId`、`Keyword`、`Page`、`Limit` | 建议可选返回元数据摘要 |
| `DescribeDocument` | `2025-11-14` | 文档详情；现有 `Metadata` 参数语义为返回内容粒度 `all/only/without` | 不复用该参数表达自定义元数据，避免语义冲突 |
| `RetrieveKnowledge` | `2025-11-14` | 知识库检索；请求体包含 `DatasetId`、`Query`、`RetrievalModel` | 建议在 `RetrievalModel` 下新增 `MetadataFilter` |
| `BatchDisplayStatus` / `DisplayStatus` / `IndexingStatus` | `2025-11-14` | 文档索引/展示状态轮询 | 元数据变更触发索引刷新时可复用批次轮询语义 |

瀚海底层已见能力：元数据定义 CRUD、文档元数据批量赋值、检索过滤条件。星流平台当前 API Explorer 尚未暴露同等能力，因此本方案定义的是“星流适配层最小新增/扩展契约”。

## 2. Action 适配总览

统一入口保持当前平台风格：

```text
POST http://aicp.api.ksyun.com/?Action=<ActionName>&Version=2025-11-14
Content-Type: application/json
```

| API ID | 建议 Action | 类型 | 是否新增 | 底层映射 | 前端对应 |
|---|---|---|---|---|---|
| META-API-001 | `CreateMetadataField` | 写 | 新增 Action | `POST /knowledge/openapi/v2/datasets/{dataset_id}/metadata` | 新建元数据字段 |
| META-API-002 | `ModifyMetadataField` | 写 | 新增 Action | `PATCH /knowledge/openapi/v2/datasets/{dataset_id}/metadata/{metadata_id}` | 编辑字段名称 |
| META-API-003 | `DeleteMetadataField` | 写 | 新增 Action | `DELETE /knowledge/openapi/v2/datasets/{dataset_id}/metadata/{metadata_id}` | 删除字段 |
| META-API-004 | `DescribeMetadataFields` | 读 | 新增 Action | `GET /knowledge/openapi/v2/datasets/{dataset_id}/metadata` | 元数据定义列表 |
| META-API-005 | `BatchModifyDocumentMetadata` | 写 | 新增 Action | `POST /knowledge/openapi/v2/datasets/{dataset_id}/documents/metadata` | 单篇/批量编辑文档元数据 |
| META-API-006 | `DescribeDocuments` 扩展 | 读 | 兼容扩展 | 星流文档列表 + 元数据摘要聚合 | 文档列表元数据摘要 |
| META-API-007 | `DescribeDocument` 扩展 | 读 | 兼容扩展 | 星流文档详情 + 元数据值聚合 | 单篇文档元数据编辑 |
| META-API-008 | `RetrieveKnowledge` 扩展 | 读/检索 | 兼容扩展 | 瀚海 `metadata_filtering_conditions` | 元数据检索过滤 |

## 3. 统一对象模型

### 3.1 MetadataField

| 字段 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| `MetadataId` | String | 是 | 字段稳定 ID。前端操作、赋值、过滤均优先使用 ID。 |
| `Name` | String | 是 | 用户可见字段名。同一 `DatasetId` 下唯一。 |
| `Type` | String enum | 是 | P0：`String`。预留：`Number`、`Boolean`、`DateTime`、`StringList`。 |
| `UsageDocumentCount` | Int | 是 | 已填写该字段的文档数；对应瀚海 `count`。 |
| `BuiltIn` | Boolean | 是 | 是否内置字段。自定义字段为 `false`。 |
| `Enabled` | Boolean | 否 | 仅内置字段需要；自定义字段默认不展示开关。 |
| `CreatedAt` / `UpdatedAt` | Long | 否 | 毫秒时间戳；用于审计，不是 P0 UI 必展示项。 |
| `CreatedBy` / `UpdatedBy` | String | 否 | 操作人；若无法返回，则操作审计中保留。 |

### 3.2 DocumentMetadataItem

| 字段 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| `MetadataId` | String | 是 | 字段 ID。 |
| `Name` | String | 否 | 字段名快照，用于展示；提交时可不传，由后端按 ID 解析。 |
| `Type` | String | 否 | 字段类型，用于前端控件和错误回显。 |
| `Value` | String | 按操作 | P0 全部按字符串处理；扩展类型再按类型校验。 |

## 4. 接口契约

### 4.1 `DescribeMetadataFields`

用途：进入“知识库设置 > 元数据定义”时查询字段列表。

请求：

| 字段 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| `DatasetId` | String | 是 | 知识库 ID。 |
| `Keyword` | String | 否 | 按字段名称模糊搜索。P0 可不做搜索，前端本地过滤。 |
| `Page` | Int | 否 | 默认 1。字段量较小可忽略分页，但接口建议保留。 |
| `Limit` | Int | 否 | 默认 100，上限 100。 |
| `IncludeBuiltIn` | Boolean | 否 | 默认 `true`。 |

响应：

| 字段 | 类型 | 说明 |
|---|---|---|
| `RequestId` | String | 请求唯一标识。 |
| `Fields` | MetadataField[] | 自定义字段和内置字段。 |
| `Total` | Int | 字段总数。 |
| `Page` / `Limit` / `HasMore` | Int/Boolean | 分页信息。 |

### 4.2 `CreateMetadataField`

用途：新建元数据字段。创建后字段出现在定义列表、文档编辑器和检索字段候选中。

请求：

| 字段 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| `DatasetId` | String | 是 | 知识库 ID。 |
| `Name` | String | 是 | 1-40 字符；trim 后不能为空；同知识库唯一。 |
| `Type` | String enum | 是 | P0 只开放 `String`；后端可先预留其他枚举但前端不展示。 |
| `ClientToken` | String | 否 | 幂等键；建议前端生成，防重复提交。 |

响应：

| 字段 | 类型 | 说明 |
|---|---|---|
| `RequestId` | String | 请求唯一标识。 |
| `Field` | MetadataField | 新建字段。 |

### 4.3 `ModifyMetadataField`

用途：编辑字段名称。P0 不允许修改类型。

请求：

| 字段 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| `DatasetId` | String | 是 | 知识库 ID。 |
| `MetadataId` | String | 是 | 字段 ID。 |
| `Name` | String | 是 | 新名称；同知识库唯一。 |
| `ClientToken` | String | 否 | 幂等键。 |

响应：

| 字段 | 类型 | 说明 |
|---|---|---|
| `RequestId` | String | 请求唯一标识。 |
| `Field` | MetadataField | 更新后的字段。 |

语义：

- 类型不可改；如请求体携带 `Type`，后端应忽略或返回 `InvalidParameter`，二选一需后端评审确定。建议返回错误，避免误解。
- 字段被检索配置引用时，允许改名，因为过滤应以 `MetadataId` 作为稳定引用；后端需要同步字段名快照。

### 4.4 `DeleteMetadataField`

用途：删除元数据字段定义。

请求：

| 字段 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| `DatasetId` | String | 是 | 知识库 ID。 |
| `MetadataId` | String | 是 | 字段 ID。 |
| `ClientToken` | String | 否 | 幂等键。 |

响应：

| 字段 | 类型 | 说明 |
|---|---|---|
| `RequestId` | String | 请求唯一标识。 |
| `Deleted` | Boolean | 是否删除成功。 |

删除规则：

- 若字段被检索配置引用，返回 `409 FieldInUse`，前端展示影响范围并阻断删除。
- 若字段仅有文档值但未被检索配置引用，允许删除；后端同步删除相关文档值，并触发索引刷新。
- 若字段不存在，返回 `404 MetadataFieldNotFound`。

### 4.5 `BatchModifyDocumentMetadata`

用途：单篇或批量编辑文档元数据值。单篇编辑也走此 Action，只传一个 `DocumentId`。

请求：

| 字段 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| `DatasetId` | String | 是 | 知识库 ID。 |
| `Operation` | String enum | 是 | `Upsert` 新增/更新值；`Clear` 清空值。 |
| `OperationData` | Object[] | 是 | 每个文档一条操作。 |
| `OperationData[].DocumentId` | String | 是 | 文档 ID。 |
| `OperationData[].MetadataList` | Object[] | 是 | 字段和值列表。 |
| `MetadataList[].MetadataId` | String | 是 | 字段 ID。 |
| `MetadataList[].Value` | String | `Upsert` 必填 | P0 String。`Clear` 时不传或传空均忽略。 |
| `ClientToken` | String | 否 | 幂等键。 |

响应：

| 字段 | 类型 | 说明 |
|---|---|---|
| `RequestId` | String | 请求唯一标识。 |
| `SucceededItems` | Object[] | 成功项：`DocumentId`、`MetadataIds[]`。 |
| `FailedItems` | Object[] | 失败项：`DocumentId`、`MetadataId`、`Code`、`Message`。 |
| `IndexBatch` | String | 触发索引刷新时返回，可复用 `BatchDisplayStatus` 轮询。 |

批量语义：

- P0 采用“部分成功”：成功项保留，失败项返回明细。这样与当前批量 UI 最匹配。
- `Operation=Clear` 是清空唯一语义；不要让 `null`、空字符串、缺失字段分别代表不同含义。
- `Operation=Upsert` 时，空字符串默认非法；如后端希望允许空字符串作为值，需要在字段规则里显式声明。

### 4.6 `DescribeDocuments` 扩展

用途：文档列表展示元数据摘要列。

请求新增可选字段：

| 字段 | 类型 | 默认 | 说明 |
|---|---|---|---|
| `IncludeCustomMetadata` | Boolean | `false` | 为 `true` 时返回元数据摘要。 |

响应 `Data[]` 新增字段：

| 字段 | 类型 | 说明 |
|---|---|---|
| `CustomMetadataSummary` | Object[] | 最多返回前 3 个字段：`MetadataId`、`Name`、`Value`。 |
| `CustomMetadataCount` | Int | 当前文档已填写字段数。 |

兼容性：默认不返回，不影响旧调用。

### 4.7 `DescribeDocument` 扩展

用途：单篇文档元数据查看/编辑抽屉。

请求新增可选字段：

| 字段 | 类型 | 默认 | 说明 |
|---|---|---|---|
| `IncludeCustomMetadata` | Boolean | `false` | 为 `true` 时返回该文档元数据值。 |

响应新增字段：

| 字段 | 类型 | 说明 |
|---|---|---|
| `CustomMetadata` | DocumentMetadataItem[] | 已填写字段和值。 |

兼容性：不要复用当前 `Metadata=all/only/without` 参数表达自定义元数据。

### 4.8 `RetrieveKnowledge` 扩展

用途：在知识库检索时按元数据过滤候选文档/切片。

请求在现有 `RetrievalModel` 下新增：

```json
{
  "DatasetId": "kb_xxx",
  "Query": "退款流程",
  "RetrievalModel": {
    "SearchMethod": "hybrid",
    "TopK": 10,
    "MetadataFilter": {
      "LogicalOperator": "and",
      "Conditions": [
        {
          "MetadataId": "meta_department",
          "FieldName": "部门",
          "Operator": "eq",
          "Value": "客服"
        }
      ]
    }
  }
}
```

字段规则：

| 字段 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| `MetadataFilter.LogicalOperator` | enum | 是 | `and` / `or`。 |
| `Conditions[].MetadataId` | String | 建议必填 | 稳定字段 ID；适配层映射到瀚海字段名。 |
| `Conditions[].FieldName` | String | 否 | 字段名快照，仅用于调试和兼容。 |
| `Conditions[].Operator` | enum | 是 | P0：`eq`、`ne`、`contains`、`in`、`not_in`。 |
| `Conditions[].Value` | String | 单值操作必填 | `eq/ne/contains` 使用。 |
| `Conditions[].Values` | String[] | 多值操作必填 | `in/not_in` 使用。 |

响应建议新增：

| 字段 | 类型 | 说明 |
|---|---|---|
| `FilterSummary` | Object | `BeforeDocumentCount`、`AfterDocumentCount`、`AppliedMetadataFilter`。 |
| `Records[].Document.CustomMetadataSummary` | Object[] | 命中文档的过滤相关元数据摘要。 |

兼容性：不传 `MetadataFilter` 时与现有 `RetrieveKnowledge` 完全一致。

## 5. 权限 Action 建议

| 权限 Action | 对应接口 |
|---|---|
| `aicp:DescribeMetadataFields` | `DescribeMetadataFields`、带元数据的 `DescribeDocuments/DescribeDocument` |
| `aicp:CreateMetadataField` | `CreateMetadataField` |
| `aicp:ModifyMetadataField` | `ModifyMetadataField` |
| `aicp:DeleteMetadataField` | `DeleteMetadataField` |
| `aicp:ModifyDocumentMetadata` | `BatchModifyDocumentMetadata` |
| `aicp:RetrieveKnowledge` | `RetrieveKnowledge` 元数据过滤沿用检索权限；字段可见性按知识库权限收敛 |

只读策略：允许查询，不允许新增、编辑、删除、赋值和过滤配置保存。

## 6. 错误模型

统一响应建议：

```json
{
  "RequestId": "req_xxx",
  "Code": "Conflict.MetadataFieldInUse",
  "Message": "元数据字段正在被检索配置引用",
  "Details": {
    "MetadataId": "meta_xxx",
    "ReferenceCount": 1
  }
}
```

| HTTP | Code | 触发 | 前端处理 |
|---:|---|---|---|
| 400 | `InvalidParameter` | 名称为空、类型不支持、值类型不匹配、条件为空 | 字段级错误，保留输入 |
| 403 | `Forbidden` | 无字段管理/文档赋值/检索权限 | 隐藏或禁用操作，弹权限提示 |
| 404 | `DatasetNotFound` / `DocumentNotFound` / `MetadataFieldNotFound` | 对象不存在或已删除 | 刷新列表，保留未提交输入 |
| 409 | `Conflict.MetadataFieldDuplicated` | 字段重名 | 定位到名称输入 |
| 409 | `Conflict.MetadataFieldInUse` | 字段被检索配置引用，删除被阻断 | 展示影响范围 |
| 409 | `Conflict.StaleVersion` | 并发修改 | 提示刷新后重试 |
| 413/400 | `LimitExceeded` | 字段数量、批量文档数或值长度超限 | 提示上限 |
| 200 | `PartialFailure` | 批量部分失败 | 展示失败清单，成功项保留 |
| 202/200 | `IndexRefreshing` | 值保存成功但索引刷新中 | 成功轻提示；仅异常/处理中时展示 |

## 7. 仍需后端拍板但不能阻塞前端评审的点

| ID | 问题 | 建议默认值 |
|---|---|---|
| META-CONFIRM-001 | 最终 Action 名称是否采用本方案 | 采用本方案，后端评审可改名但语义不变 |
| META-CONFIRM-002 | P0 是否只开放 `String` | 是，其他类型后续扩展 |
| META-CONFIRM-003 | 字段数量/名称长度上限 | 字段 100 个；名称 1-40 字符；值 0-500 字符 |
| META-CONFIRM-004 | 删除已填写值但未被检索引用的字段是否允许 | 允许，删除字段和文档值，并触发索引刷新 |
| META-CONFIRM-005 | 批量最大文档数 | 100，与现有导入/分页量级保持一致 |
