# 错误码：元数据

状态：部分 HTTP 状态已在 Apifox 看到；业务 code/错误体待确认

| 场景 | HTTP/业务码 | 触发条件 | 前端提示 | 恢复方式 | 状态 |
|---|---|---|---|---|---|
| 请求参数错误 | 400 / TO_CONFIRM | 字段名非法、类型错误、value 不符合类型、空值语义错误 | “请检查元数据字段和值”并定位字段 | 修改输入后重试 | PARTIAL：HTTP 已见 |
| 未登录/密钥无效 | 401 / TO_CONFIRM | 登录态或 API Key 无效 | “登录已失效，请重新登录” | 重新登录 | PARTIAL：HTTP 已见 |
| 无权限 | 403 / TO_CONFIRM | 无知识库或文档编辑权限 | “你没有操作该元数据的权限” | 申请权限/切换账号 | PARTIAL：HTTP 已见 |
| 知识库/文档/字段不存在 | 404 / TO_CONFIRM | dataset、document、metadata_id 不存在 | “对象不存在或已被删除，请刷新” | 刷新页面 | PARTIAL：HTTP 已见 |
| 字段重名 | 400/409 / TO_CONFIRM | 同一知识库下字段名重复 | “元数据字段名称已存在” | 修改名称 | TO_CONFIRM |
| 字段数量超限 | 400 / TO_CONFIRM | 元数据定义数量超过限制 | “元数据字段数量已达上限” | 删除无用字段 | TO_CONFIRM |
| 字段被引用无法删除 | 409 / TO_CONFIRM | 字段被检索配置或规则引用 | “该字段正在被使用，解除引用后再删除” | 查看引用并解除 | TO_CONFIRM |
| 批量部分失败 | 207/200+failed_items / TO_CONFIRM | 部分文档更新失败 | “部分文档更新失败，请查看失败清单” | 重试失败项 | TO_CONFIRM |
| 索引/检索生效延迟 | 202/200+pending / TO_CONFIRM | 更新成功但过滤索引未生效 | “元数据已保存，检索生效中” | 等待/刷新/重试召回测试 | TO_CONFIRM |
| 并发冲突 | 409 / TO_CONFIRM | 字段或文档被他人同时修改 | “数据已变化，请刷新后重试” | 刷新页面 | TO_CONFIRM |
