# 错误码：知识库新增元数据

状态：评审稿。当前平台公开文档只列通用错误区，以下为元数据接口建议业务错误模型，后端可调整 Code 命名但需保留语义。

统一错误体：

```json
{
  "RequestId": "req_xxx",
  "Code": "Conflict.MetadataFieldInUse",
  "Message": "元数据字段正在被检索配置引用",
  "Details": {}
}
```

| HTTP | Code | 触发条件 | 前端提示 | 恢复方式 |
|---:|---|---|---|---|
| 400 | `InvalidParameter` | 名称为空、类型不支持、条件为空、值类型不匹配 | “请检查填写内容”并定位字段 | 用户修改后重试 |
| 400 | `InvalidMetadataValue` | `Operation=Upsert` 但值为空或格式错误 | “字段值格式不正确” | 修改该字段值 |
| 403 | `Forbidden` | 无字段管理、文档赋值或检索过滤权限 | “你没有操作该元数据的权限” | 申请权限或切换账号 |
| 404 | `DatasetNotFound` | 知识库不存在或无权访问 | “知识库不存在或无权限” | 刷新/返回列表 |
| 404 | `DocumentNotFound` | 文档不存在或已删除 | “文档不存在或已被删除” | 刷新文档列表 |
| 404 | `MetadataFieldNotFound` | 字段不存在、已删除或不属于该知识库 | “元数据字段不存在，请刷新” | 刷新字段列表 |
| 409 | `Conflict.MetadataFieldDuplicated` | 同知识库字段重名 | “元数据字段名称已存在” | 修改名称 |
| 409 | `Conflict.MetadataFieldInUse` | 删除字段时被检索配置引用 | “该字段正在被检索条件使用，需先移除引用” | 移除引用后重试 |
| 409 | `Conflict.StaleVersion` | 并发修改导致版本过期 | “数据已变化，请刷新后重试” | 刷新后重试 |
| 413/400 | `LimitExceeded` | 字段数、批量文档数、值长度超限 | “已超过数量限制” | 减少数量或缩短内容 |
| 200 | `PartialFailure` | 批量编辑部分文档失败 | “部分文档更新失败” | 展示失败清单，允许重试失败项 |
| 202/200 | `IndexRefreshing` | 保存成功但索引刷新中 | “元数据已保存，检索生效中” | 等待或刷新；成功态不常驻展示 |
| 500/503 | `IndexRefreshFailed` | 索引刷新失败 | “元数据已保存，但检索暂未生效” | 重试刷新或联系管理员 |

## 批量失败明细建议

```json
{
  "SucceededItems": [
    {"DocumentId": "doc_1", "MetadataIds": ["meta_department"]}
  ],
  "FailedItems": [
    {
      "DocumentId": "doc_2",
      "MetadataId": "meta_department",
      "Code": "DocumentNotFound",
      "Message": "文档不存在或已被删除"
    }
  ]
}
```

前端验收：批量部分失败时，成功项保留；失败项展示文档、字段和原因。
