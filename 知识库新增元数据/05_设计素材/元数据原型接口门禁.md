# 元数据原型接口门禁

更新时间：2026-07-10  
状态：P0、P1 HTML 交互原型已完成；具体星流适配字段仍以接口评审为准，P2 不纳入本版。

## 已验证

- 瀚海 OPEN-API 已有元数据定义 CRUD、文档元数据赋值、检索元数据过滤能力。
- 金山云当前 API Explorer / 星流平台知识库接口尚未暴露同等接口。
- 当前平台 `DescribeDocument.Metadata` 只是返回内容粒度，不是自定义元数据字段。

## 原型约束

| 区域 | 能画什么 | 不能画什么 |
|---|---|---|
| 新建知识库 | 完全保持现有页面和创建接口 | 不增加元数据定义或文档元数据值 |
| 知识库设置 | 元数据字段定义列表、新增字段、编辑名称、删除确认 | 不把字段类型编辑画成已支持；更新接口当前只见 `name` |
| 导入文档/文档信息 | 导入页保持现状；导入后查看/编辑单篇值和批量编辑 | 不在导入请求中增加元数据值；不与文档级标签混用同一弹窗 |
| 检索配置 | 可在 PRD 中描述“元数据过滤需要接入瀚海检索字段” | 当前不做知识问答页原型；不写成当前 `RetrieveKnowledge` 已支持 |

## 接口评审约束

- PRD 字段映射引用 `06_接口/接口清单.md` 和 `06_接口/前后端字段映射.md`。
- Pencil 文件建议单独命名为 `kb-metadata-prototype.pen`，不要使用早期 `kb-tags-metadata-light-prototype.pen`。
