# 产品定义：知识库新增元数据

状态：范围初稿，待接口字段和用户入口确认

## 1. 问题

底层已有文档元数据能力，但控制台没有入口。管理员无法可视化定义、
查看或修改文档的自定义原始信息，应用也难以稳定配置基于元数据的
检索过滤。

## 2. 产品对象

- 元数据定义：可用 key 的名称、标识、类型、约束和说明。
- 文档元数据值：某篇文档对一个元数据定义的具体 value。
- 元数据过滤条件：检索请求中用于缩小候选文档/切片范围的表达式。

这三者暂不假设使用同一接口或同一生命周期。

## 3. 用户场景

- 知识库管理员定义可用元数据，如“来源 URL”“部门”“发布日期”。
- 管理员在文档详情或批量操作中填写/修改元数据值。
- 应用开发者在检索配置中选择元数据条件。
- 系统先按条件过滤候选文档/切片，再执行召回。

## 4. 当前范围

### P0 候选

- 查看、新增、编辑、删除元数据定义。
- 在文档上查看和更新元数据值。
- 前端字段与底层接口字段一一映射。
- 输入校验、重复/不存在/无权限等错误反馈。
- 元数据条件参与检索过滤（接口与运算符待确认）。

### P1 候选

- 批量设置、清空、导入导出元数据值。
- 使用量、引用关系和删除影响提示。
- 多类型值与复杂过滤组合。

### 非目标

- 用元数据替代标签、权限或计费体系。
- 在接口证据不足时自行发明字段类型或过滤语法。

## 5. 成功信号（待基线）

- 管理员能在控台完成定义和文档赋值，不依赖手工调用接口。
- 控台提交字段与后端请求一致，错误可定位到具体文档/key。
- 检索条件能稳定排除不符合元数据的文档切片。
- 元数据更新在约定时间内对检索生效。
