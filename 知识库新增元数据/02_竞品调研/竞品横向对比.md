# 竞品横向对比：知识库新增元数据

状态：初版（公开官方文档证据 + 我方已知 Apifox 证据）

## 1. 快速结论

- 元数据的行业做法比较一致：知识库/数据集层面管理字段定义，文档/文件层面填写值，检索前按 metadata filter 缩小候选范围。
- Dify 的交互最接近我方需求：Metadata Panel 管字段定义，文档列表批量编辑，文档详情单独编辑，检索节点启用 metadata filtering。
- RAGFlow 也明确区分 dataset 级和 file 级 metadata；它的“重名值自动合并”和“删除字段影响全局关联文件”值得作为边界提醒。
- 阿里云 Model Studio/PAI 强调 metadata 在 chunk/文件过滤、召回测试和应用流程中的使用，说明 PRD 不能只写管理入口，必须写检索配置和验证入口。
- 我方已有底层接口，PRD P0 应优先补控制台入口、字段映射、错误反馈和检索过滤，而不是重新发明数据模型。

## 2. 横向对比表

| 维度 | 我方目标 | Dify | RAGFlow | 阿里云 Model Studio/PAI | 结论状态 |
|---|---|---|---|---|---|
| 元数据定义层级 | 明确字段定义是否是知识库级 schema | Metadata fields 在知识库中管理，字段变更影响全知识库 | 支持 dataset 级 metadata 管理 | 知识库创建/配置中可开启 metadata 抽取/模板 | VERIFIED；我方层级待确认 |
| 字段类型 | 支持文本、数字、时间等类型，类型是否可改 | API 类型包含 `string/number/time`；产品文档说明字段类型创建后不可改 | 公开页面未展开类型细节 | 支持 Constant/Variable/LLM/RegEx 等抽取方式 | VERIFIED / TO_CONFIRM |
| 内置字段 | 是否支持来源 URL、文件名等系统字段 | 区分 built-in 与 custom metadata；built-in 默认可开关/系统生成 | 待控制台补证 | 文档提到 file_name、cat_name 等变量 | VERIFIED / TO_CONFIRM |
| 批量文档赋值 | 文档列表多选后批量编辑 | 支持批量编辑；可选择更新范围 | 待补证 | PAI 可在文档 chunk 详情编辑后做召回测试 | VERIFIED / TO_CONFIRM |
| 单文档赋值 | 文档详情可编辑该文档 metadata | 支持文档详情页编辑 | 支持 file details 中 Set Metadata | PAI 在文档 chunk 详情编辑 metadata | VERIFIED |
| 删除影响 | 删除定义和值需要区分影响范围 | Metadata Panel 删除字段影响全知识库；编辑器/详情页删除只影响选中文档/当前文档 | 删除字段或值会影响关联文件；重名值可能合并 | 待补证 | VERIFIED |
| 检索过滤 | 应用/召回测试可配置 metadata 条件 | Knowledge Retrieval 可启用 metadata filtering | 待补证 | API 可传 `metadata_filter`；Recall Test 可加条件 | VERIFIED |
| 接口能力 | 控台字段需映射到底层 API | API 覆盖 metadata field CRUD、built-in toggle、batch update document metadata | HTTP API 待深入 | 文档显示 API 参数，但控制台/接口细节待进一步采集 | VERIFIED / TO_CONFIRM |
| 控台入口 | 知识库设置 + 文档详情/列表 + 召回测试 | 三个入口都比较明确 | dataset + file details | 知识库配置、chunk 详情、Recall Test、应用流程 | VERIFIED |
| 风险点 | 删除/改名/批量/索引生效要可解释 | 全局字段删除风险高；类型不可改 | 重名值自动合并可能造成数据治理风险 | 创建后部分 metadata extraction 配置不可改 | VERIFIED / INFERRED |

## 3. 对我方 PRD 的建议

1. P0 拆成三段：定义管理、文档赋值、检索过滤。
2. 定义管理入口建议放在知识库设置或知识库详情的“元数据”页；文档赋值入口放在文档详情和文档列表批量操作。
3. 删除必须区分“删除字段定义”和“删除某文档上的字段值”，两者影响范围完全不同。
4. 检索过滤必须写进 PRD 和接口验收；只做控台管理会失去元数据价值。
5. 批量更新需要明确 `partial_update`/覆盖/清空语义；我方内部接口是否支持仍待确认。
