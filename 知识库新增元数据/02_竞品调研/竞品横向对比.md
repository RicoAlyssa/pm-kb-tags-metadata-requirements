# 竞品横向对比：知识库新增元数据

状态：初版（正式竞品已收敛为火山云、阿里云；控制台截图待采）

## 1. 范围说明

正式竞品只包含：

- 火山云/火山引擎：VikingDB、Viking 知识库、火山方舟/火山知识库相关链路。
- 阿里云：百炼 Model Studio、PAI 知识库相关链路。

Dify、RAGFlow、AWS 等实现只保留为参考实现，不进入竞品横向结论。

## 2. 快速结论

- 阿里云是元数据需求的核心正式竞品：公开文档说明可提取元数据、查看文件 Meta 信息、在向量检索前进行结构化搜索，并提供 SearchFilters / MetaDataFilterConditions API。
- 火山云更像“标签字段 + 文档信息更新 + 检索过滤”的实现：官方 API 列表存在 `update_meta`，火山知识库官方文档支持标签配置和按标签过滤文档范围；具体过滤字段名仍待 Network/HAR 确认。
- 两家正式竞品都证明：元数据/标签字段不是单纯展示信息，必须进入“检索前过滤”链路。
- 我方 PRD 仍需以底层 Apifox 为准：竞品字段名、运算符、过滤表达式都不能直接搬进我方接口。

## 3. 横向对比表

| 维度 | 我方目标 | 火山云/火山引擎 | 阿里云百炼/PAI | 产品判断 |
|---|---|---|---|---|
| 元数据/字段定义 | 明确字段定义层级、类型、限制 | 公开证据包含 `update_meta` 与标签字段配置；完整 schema 待 API/控制台补证 | 百炼强调元数据提取，PAI 返回 chunk metadata；字段定义入口待控制台补证 | 我方定义管理仍需以后端 Apifox 为准 |
| 文档赋值/更新 | 单文档与批量文档填写、清空、覆盖 | 火山可导入时传字段或上传后更新文档信息；UI 可在原始文档标签列编辑标签 | 百炼数据管理列表可查看 Meta 信息；PAI API 返回 chunk metadata | 我方 P0 应包含文档详情和批量编辑 |
| 检索过滤 | 元数据条件进入检索前过滤 | 火山知识检索参数支持按标签过滤文档范围，条件含包含/不包含/大于/小于等；字段名待 HAR 确认 | 百炼 SearchFilters 过滤召回文本切片；PAI `MetaDataFilterConditions` 支持 `and/or` 与 `==/!=/contains` | 检索过滤必须写入 PRD 和验收 |
| 运算符/组合逻辑 | 支持哪些 operator、AND/OR、空值语义 | 公开证据暂未展开完整 operator | PAI 明确 `FilterCondition=and/or`，operator 包含 `==`、`!=`、`contains`（仅 file_name） | 我方接口需要至少确认等于/不等于/包含/范围是否支持 |
| 控台入口 | 知识库设置、文档详情/列表、召回测试 | 火山：创建知识库高级配置、原始文档标签列、检索/问答 | 阿里：数据管理、Meta 信息、SearchFilters、召回/检索服务 | Pencil 原型需覆盖定义、赋值、过滤三入口 |
| 删除/改名影响 | 删除字段定义与删除文档值要分开 | 火山创建后标签名不可变更是重要风险参考 | 阿里公开文档未充分说明删除影响 | 我方必须明确影响范围和引用阻断 |
| 计费/资源 | 元数据本身不计费，但检索/存储/索引可能产生资源费用 | 火山知识库资源本身计费，文档删除不一定释放计算资源 | 阿里知识库商业化和配额另需补证 | 元数据不进入计费，但要提示索引/检索资源影响 |

## 5. 对我方 PRD 的建议

1. 元数据必须同时覆盖“定义管理、文档赋值、检索过滤”三段，否则功能价值不闭环。
2. 过滤表达式是当前最大接口门禁：阿里云 PAI 有明确 operator 参考，但我方必须以后端接口为准。
3. 火山“创建后标签名不可变更”说明字段变更风险很高；我方若允许改名，需要写明索引重建、生效时间和引用影响。
4. Dify/RAGFlow 继续放在参考实现池，可启发交互，但不能作为竞品结论或优先级依据。
