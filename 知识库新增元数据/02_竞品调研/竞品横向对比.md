# 竞品横向对比：知识库元数据

状态：已完成（2026-07-13）

正式竞品：火山方舟、阿里云百炼/PAI。

## 1. 前端控制台对比

| 流程/页面 | 火山方舟旗舰版 | 阿里云百炼/PAI | 金山云取舍 | 截图证据 |

## 2. 后端接口对比

| 流程 | 火山方舟 | 阿里云百炼/PAI | 金山云最小改造 |
| --- | --- | --- | --- |
| 创建知识库 | POST .../CreateCollection；旗舰版 UI 已确认强类型 Schema | POST .../indices/rag/index/create，携带 metaExtractColumns[] | 不修改现有创建主接口；知识库创建后单独调用元数据定义接口 |
| 定义管理 | Schema 含名称、类型、可选项；名称创建后不可改 | metaExtractColumns：key、desc、type、value、enableSearch、enableLlm | 直接适配瀚海元数据 CRUD；P0 使用 name/type，仅支持 String |
| 文档导入 | POST .../AddDoc | dataHub.createDataListReturnFileIds | 不修改 ImportDocuments；文档导入完成后通过独立接口单篇或批量赋值 |
| 单篇/批量赋值 | 官方 update_meta 与单篇编辑入口已确认 | 抽取链路生成 Meta 键值 | 复用瀚海 operation_data[].document_id + metadata_list[] 批量更新 |
| 文档查询 | GetKnowledgeDocList、ListDocs、ListChunks | rag-knowledgebase.dataList/detail | DescribeDocuments/DescribeDocument 可选返回元数据摘要 |
| 保存过滤 | POST .../CreateSearchConfig，条件含字段名、类型、操作符和值 | POST .../rag/app/update，写入 agent_config | RetrieveKnowledge 增加可选 metadata_filtering_conditions |
| 执行检索 | GetKnowledgeServiceChat | POST .../indices/knowledge/search | 复用瀚海 logical_operator + conditions[]，旧请求不传时不改变召回 |
| 运算符 | 按字段类型变化；实测 List-string 的 Operator 0 表示包含 | 百炼支持单值、多值、范围、like；PAI 支持等于、不等于、包含 | P0 String：等于、不等于、包含；非法组合由后端白名单拦截 |
| 内置字段 | doc_id 默认可过滤 | 平台字段参与过滤 | 内置字段只读返回；P0 不开放删除/改名 |
| 生效状态 | 通过检索结果验证 | 随索引任务/发布生效 | 更新后异步刷新索引，统一返回处理中/成功/失败与失败原因 |

## 3. 实测请求结构

阿里创建字段结构：

```json
{"metaExtractColumns":[{"key":"evidence_key","desc":"采证字段","type":"constant","value":"codex_20260712","enableSearch":true,"enableLlm":false}]}
```

火山检索字段结构：

```json
{"Filter":{"Relation":0,"Conditions":[{"FieldName":"liststring","FieldType":"list<string>","Operator":0,"Values":["product"]}]},"FilterMode":"tag"}
```
