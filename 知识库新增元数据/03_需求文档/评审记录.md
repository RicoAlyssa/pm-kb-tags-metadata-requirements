# 评审记录：知识库新增元数据

## 2026-07-14 v0.5 用户当前版同步审查

结论：当前 PRD 可进入产品/前后端/QA 初评，但不能作为最终排期闭环。已同步用户当前 PRD 中的关键边界：创建知识库页和创建知识库主接口不增加元数据定义；元数据定义放在知识库设置中维护；文档元数据在导入完成后通过文档信息单篇/批量赋值。

| 维度 | 审查结论 | 后续动作 |
|---|---|---|
| 创建链路 | 已废弃旧决策 `META-DEC-005`，新增 `META-DEC-006`：创建知识库不预定义元数据 | 原型和接口评审时不得把元数据定义放回创建页 |
| 定义层级 | 元数据定义是知识库级 Schema，文档只填写已定义字段的值 | 后端需确认 `name/type/id/count`、类型枚举、名称规则、数量限制 |
| 文档赋值 | 导入文档页不填写元数据；导入完成后在文档信息页单篇/批量维护 | 前端需明确清空值、覆盖值、部分更新、批量失败的交互 |
| 接口 | 瀚海已有定义 CRUD、文档元数据更新、retrieve filter 线索；星流平台尚未暴露同等能力 | 后端需确认星流适配层 Action 名称、版本、错误码和权限 |
| 检索 | 元数据用于检索前过滤候选文档/切片；旧请求不传 filter 时行为不变 | 检索负责人需确认条件组、AND/OR、运算符、空值和索引生效延迟 |
| 设计 | 现有 HTML 最小改造原型可作为评审讨论稿；正式 Pencil 原型仍待最终确认 | 原型必须与标签需求分开，不在同一张图里混合呈现 |

### 当前阻塞项

1. 星流适配层是否直接复用瀚海路径，还是封装成星流 `Action + Version` 风格，仍待后端确认。
2. 元数据类型枚举、运算符枚举、名称/数量/长度限制尚未最终确认。
3. 批量更新语义（部分更新、覆盖、清空、部分成功）尚未形成接口契约。
4. 内置字段 `built_in_field_enabled` 是否在本期开放、是否只读、是否可开关仍待确认。
