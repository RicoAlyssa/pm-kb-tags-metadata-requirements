- 当前状态：已按用户确认将正式竞品收敛为火山云和阿里云；Dify/RAGFlow/AWS 等仅作参考实现；已基于金山云当前知识库详情页制作轻量 HTML 原型草稿；PRD 已同步到 v0.5；后端接口已按金山云 API Explorer 的 `Action + Version=2025-11-14` 风格补齐最小改动契约，明确创建知识库页和主接口不增加元数据定义，定义在知识库设置维护，文档值在导入完成后赋值

## 范围

### 已确认

- 元数据是文档级可扩展原始信息，可保存 URL 或客户自定义关键 key 信息。
- 检索时可携带元数据条件筛选文档/切片。
- 底层已有新增、更新、删除、查询元数据及更新文档元数据接口。
- 当前重点是将能力暴露到控制台，并明确前端入口、字段与后端映射。

### 非目标

- 不把元数据与标签合并成同一产品对象。
- 不在未验证时假设字段类型、唯一性、限额或检索运算符。

## 2026-07-10 接口门禁更新

- VERIFIED：瀚海 OPEN-API 已有元数据定义 CRUD、文档元数据赋值和检索元数据过滤相关接口。
- VERIFIED：金山云当前 API Explorer / 星流平台知识库接口未暴露同等元数据定义 CRUD、文档元数据赋值接口；`DescribeDocument.Metadata` 仅表示返回内容粒度 `all / only / without`，不是自定义元数据字段。
- INFERRED：用户判断“瀚海 API 做了但没上线到我们平台”与现有证据一致；更准确口径是“底层能力已在瀚海 OPEN-API 出现，但尚未体现在当前星流平台开放接口/控制台”，是否灰度或内部适配仍需后端确认。
- DECISION：正式元数据原型不得与标签原型合并；需先按“瀚海能力如何接入星流平台”的接口约束设计。
- IMPACT：之前的“标签+元数据轻量合并 HTML 草稿”仅保留为历史草稿，不再作为正式原型依据。

## 2026-07-14 后端接口契约补齐

- VERIFIED：已按当前星流知识库接口风格补齐元数据最小 Action 契约，不再只列缺口。
- VERIFIED：`DescribeDocument.Metadata` 与自定义元数据冲突，不能复用；本版改用 `IncludeCustomMetadata` / `CustomMetadata` 扩展文档列表与详情。
- DECISION：检索侧最小改动是在现有 `RetrieveKnowledge.RetrievalModel` 内新增 `MetadataFilter`，而不是新增一套检索 Action。
